# Security policy

Report suspected vulnerabilities privately through [GitHub private vulnerability reporting](https://github.com/CruxExperts/envman/security/advisories/new). Do not include secrets, encrypted-backup passwords, or production variable values in public issues.

## Scope

This repository covers Envman's source, release installer, published release assets, and GitHub Actions workflows. The local `uv` executable, Python installation, shell configuration, GitHub account, PyPI, and your machine's environment are external trust boundaries. See [the installation reference](docs/reference/install-source-and-updates.md).

We will acknowledge a private report, assess reproducibility and impact, and coordinate a fix before disclosure where appropriate.
