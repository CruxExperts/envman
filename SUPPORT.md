# Support

## Choose a channel

- Ask usage, configuration, and design questions in [GitHub Discussions](https://github.com/CruxExperts/envman/discussions).
- Report a reproducible defect in [GitHub Issues](https://github.com/CruxExperts/envman/issues).
- Report a suspected vulnerability through [private vulnerability reporting](https://github.com/CruxExperts/envman/security/advisories/new), as described in [SECURITY.md](SECURITY.md).

## Include useful diagnostics

For a bug or configuration question, include:

- Envman version from `envman --version`
- Linux distribution, Python version, and `uv` version
- The exact Envman command or TUI action
- Reproduction steps, expected behavior, actual behavior, and sanitized output

Remove environment values, API keys, backup passwords, tokens, private paths, hostnames, and other machine-specific details before posting. If a report involves an installer or update, include the public release version and the sanitized error text, but do not attach a receipt or backup file unless it has been scrubbed.
